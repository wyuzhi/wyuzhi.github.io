###远程操作

####以example为例，先在github新建仓库

####接下来的操作，很明显github都告诉你了

##本地操作

####git init 本地建立git仓库

####git add . 将文件传至暂存区

####git commit -m “first commit” 将文件传至git本地仓库

####git remote add origin git@github.com:wyuzhi/example.git 将本地仓库与远程仓库连接

####git push -u origin master 将本地仓库的文件传入远程仓库

###进阶操作

·创建一个空文件夹，”git init” 初始化仓库；

·文件夹中创建一个文本文件（如：zhi.txt)：”git add zhi.txt” 将文件添加到git 仓库的缓存区；
(说明：git add 命令是将内容添加至缓存区，即可以添加多个文件，然后使用git commit命令 将多个文件同时提交)

·” git commit –m “对提交文件的说明” “:
(简单解释一下git commit命令，-m后面输入的是本次提交的说明，可以输入任意内容 ，当然最好是有意义的，这样你就能从历史记录里方便地找到改动记录。-廖雪峰)

·”git log” 查看日志(更多git log内容：https://www.cnblogs.com/irocker/p/git- log.html）
($ git log
commit d95e902addd95e164bfd172fb327fe6a3705bc75 (HEAD -> master)
Author: xxx
Date: Tue Dec 26 08:54:34 2017 +0800
xiugai
commit 7f0732172eac887d6aefc8bb359190589b5868b8
Author: xxx
Date: Tue Dec 26 08:31:44 2017 +0800
yanshi
日志呈倒叙，即最近修改的在上；)

·”git reflog” 可以查看所有的commit 的日志:
($ git reflog
d95e902 (HEAD -> master) HEAD@{0}: commit: xiugai
7f07321 HEAD@{1}: commit (initial): yanshi )

·修改文件（zhi.txt）后，”git diff”/“git diff zhi.txt” 可以查看修改内容:
($ git diff
diff –git a/zhi.txt b/zhi.txt
index 99e0e11..db27b31 100644
— a/zhi.txt
+++ b/zhi.txt
@@ -1,2 +1,3 @@
Git is a distributed version control system.
-Git is free software distributed under the GPL.
\ No newline at end of file
+Git is free software distributed under the GPL.
+i like git
\ No newline at end of file )

·”git status”命令可以查看仓库当前的状态
($ git status
On branch master //master分支
Changes not staged for commit: //未提交的更改：
(use “git add …” to update what will be committed) //（使用Git添加<文件>……）更新所提交的内容）
(use “git checkout – …” to discard changes in working directory) //（使用Git结帐- <文件>……）删除工作目录中的更改） 

modified:   zhi.txt                                                          //修改：zhi.txt

no changes added to commit (use “git add” and/or “git commit -a”) ) //没有更改添加到提交（使用“Git添加”和/或“Git提交- A”）

·版本回退 命令” git reset –hard HEAD^ “(“^”的个数代表回退版本数：上一个 版 本就是HEAD^，上上一个版本就是HEAD^^）
（$ git reset –hard HEAD^
HEAD is now at d95e902 xiugai）

·回到未来（上面各种查看日志命令后得到的commit码便尤为重要）也有更简洁的日志体 现即：” git log –pretty=oneline “
($ git log –pretty=oneline
d95e902addd95e164bfd172fb327fe6a3705bc75 (HEAD -> master) xiugai
7f0732172eac887d6aefc8bb359190589b5868b8 yanshi)
如上这一串字母与数字的组合就是commit id；
可见回退版本后现在只有两个版本，但又想回到未退回之前的版本即未来；
就需要那个版本的commit id 使用命令：

·” git reset –hard commit id “ 即可回到未来甚至任何一个版本。
但如我现在忘记了未来的版本号，也莫慌：

· 命令” git reflog “ 记录你的每一次命令：
（$ git reflog
d95e902 (HEAD -> master) HEAD@{0}: reset: moving to HEAD^
4f2234d HEAD@{1}: commit: like
d95e902 (HEAD -> master) HEAD@{2}: commit: xiugai
7f07321 HEAD@{3}: commit (initial): yanshi）

·本地仓库连接远程仓库 “ git remote add origin SSH地址 ”
